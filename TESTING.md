# Тестирование WeatherSync

Проверки автоматизированы. Есть три способа запуска.

> **Тесты выключены по умолчанию.** Чтобы включить, установите в
> [config.lua](config.lua) `Config.enableTests = true` и перезапустите ресурс.
> На проде держите флаг выключенным — команды `/weathertest` и `weathertest`
> просто не регистрируются. Диагностика `/synccheck` работает всегда,
> независимо от флага.

## 1. В игре: `/weathertest` (любой игрок)

Read-only набор, ~15 секунд, ничего не меняет на сервере:

| Тест | Что проверяет |
|------|---------------|
| T1 | Конвертация времени `TimeToDHMS`/`DHMSToTime` (roundtrip) |
| T2 | Стороны света по градусам ветра |
| T3 | Региональный перевод погоды на фиксированных координатах (снежный регион, пустыня, Гуарма) |
| T4 | Что `init()` выполнился и состояние от сервера получено |
| T5a-g | Полное сравнение с эталонным состоянием сервера: время, внутриигровые часы, погода (получена и применена по региону), ветер, timescale, флаг заморозки |
| T6 | Что часы реально идут с нужной скоростью (замер за 4 секунды, вычисленное время и фактические игровые часы) |
| T7 | Отсутствие сетевого трафика в простое (5 секунд наблюдения за счётчиками событий) |

Результаты — в чат и в консоль F8 (`[weathertest] ...`). В конце сводка:
`done: N passed, 0 failed, 0 skipped`.

## 2. В игре: `/weathertest full` (нужны admin-права)

Всё из read-only плюс мутирующие тесты (~40 секунд). Состояние сервера
**восстанавливается** после каждого теста (время — с компенсацией потраченных
на тест секунд):

| Тест | Что проверяет |
|------|---------------|
| T8 | Смена погоды сервером доходит до клиента и применяется с учётом региона |
| T9 | Установка времени: событие доходит, внутриигровые часы реально переставляются |
| T10 | Смена timescale: скорость хода часов меняется, **без прыжка времени** в момент переключения |
| T11 | Смена ветра доходит и применяется |

Требуются ace-права `command.weather`, `command.time`, `command.timescale`,
`command.wind`. Без них T8–T11 будут помечены `SKIP` с пояснением.

Нюанс: флаги заморозки погоды/ветра при восстановлении сбрасываются в
«не заморожено» (клиент их не знает) — об этом будет строка `Note` в конце.

## 3. В консоли сервера: `weathertest`

```
weathertest              # read-only серверные тесты (~5с)
weathertest full         # + мутирующие тесты с точным восстановлением (~10с)
weathertest client 1     # запустить клиентский набор на игроке с id 1,
                         # результаты печатаются в эту же консоль
weathertest client 1 full
```

Серверный набор:

| Тест | Что проверяет |
|------|---------------|
| S1–S2 | Конвертация времени, стороны света |
| S3 | Активный weather pattern: суммы вероятностей = 100, все ссылки существуют |
| S4 | Структура прогноза: длина, валидность типов погоды, первая запись = текущая погода |
| S5 | `nextWeather` порождает только известные типы погоды |
| S6 | Время на сервере идёт с настроенной скоростью (замер за 3 секунды) |
| S7–S10 (full) | Установка/восстановление времени, timescale, погоды, ветра. В отличие от клиентского набора, флаги заморозки восстанавливаются точно |

Мутирующие серверные тесты рассылают изменения игрокам (на пару секунд у всех
сменится погода/время) — запускайте `full` не в прайм-тайм.

## Точечная диагностика: `/synccheck`

Одноразовое сравнение клиента с сервером (то же, что T5, но с подробным
выводом каждой строки). Полезно, когда игрок жалуется на рассинхрон:
попросите его выполнить `/synccheck` и прислать скриншот, либо выполните
`weathertest client <id>` из консоли.

## Что осталось ручным

Автоматизировать нельзя — нужны два клиента или перемещение игрока:

1. **Поздний вход**: подождите 10+ минут после старта сервера, переподключитесь,
   сразу `/weathertest` — T5a должен быть `OK` (раньше время отставало на
   всё время работы сервера).
2. **Согласованность между игроками**: два клиента одновременно `/synccheck` —
   у обоих `OK` и одинаковое серверное время.
3. **Визуальный региональный переход**: `/weather rain 2`, телепорт в Ambarino
   (например `-1350, 2400, 300`) — в течение 5 секунд дождь сменится снегом
   (лог F8 при включённом `/weatherdebug`: `Applying weather: rain -> snow`).
   Логика перевода при этом покрыта автотестом T3.
4. **Права доступа**: с аккаунта без admin-прав `/weathertest full` — T8–T11
   должны дать `SKIP` (события отклонены сервером), а в серверной консоли
   появятся строки `[warning] Player N tried to use ... without permission`.
5. **Локальные переопределения**: `/weathersync` off → `/myweather snow 2 1` →
   `/weathersync` on — погода/время вернулись к серверным, затем `/weathertest`
   — всё `OK`.

## Отладочные логи

- `/weatherdebug` (клиент, вывод в F8) — события синхронизации и локальные применения погоды/ветра.
- `weatherdebug_sv` (серверная консоль) — рассылки, смены погоды, инициализации игроков.
- `/weatherstatus` (клиент) и `weatherstats` (серверная консоль) — текущее состояние и счётчики.

Здоровый сервер в простое: одна строка в лог на игровой час
(`Weather change ... broadcasting` либо `Weather tick - no change`) и
`Player N initialized weather sync` при входах. Ничего чаще.
